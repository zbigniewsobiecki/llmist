Let's run your first AI agent with gadgets from the command line.

## Your First Completion

The simplest way to use llmist is the `complete` command:

<Tabs>
  <TabItem label="bun">
    ```bash
    bunx @llmist/cli complete "What is the capital of France?"
    ```
  </TabItem>
  <TabItem label="npm">
    ```bash
    npx @llmist/cli complete "What is the capital of France?"
    ```
  </TabItem>
</Tabs>

You'll get a streaming response:

```
The capital of France is Paris.
```

## Running an Interactive Agent

The `agent` command runs an AI agent that can use tools (gadgets):

<Tabs>
  <TabItem label="bun">
    ```bash
    bunx @llmist/cli agent "What time is it in Tokyo?"
    ```
  </TabItem>
  <TabItem label="npm">
    ```bash
    npx @llmist/cli agent "What time is it in Tokyo?"
    ```
  </TabItem>
</Tabs>

When running in a terminal (both stdin and stdout are TTY), the agent automatically uses TUI mode with an interactive interface featuring expandable blocks, keyboard navigation, and live streaming.

<Aside type="tip">
  TUI mode activates automatically in interactive terminals. Use `--quiet` to disable it for scripting.
</Aside>

## Built-in Gadgets

The CLI includes three built-in gadgets for agent-user communication:

| Gadget | Purpose |
|--------|---------|
| **AskUser** | Ask the user a question and wait for their response |
| **TellUser** | Display important messages (info, success, warning, error) |
| **Finish** | Signal that the agent has completed its task |

These are registered automatically. Use `--no-builtins` to disable them.

<Aside type="note">
  For filesystem, shell, or web capabilities, create custom gadgets or use the library API.
  See [Creating Gadgets](/cli/gadgets/local-gadgets/) for examples.
</Aside>

## Creating a Local Gadget

Create a custom gadget for your agent:

<Steps>

1. **Create a gadgets directory**

   ```bash
   mkdir gadgets
   ```

2. **Create a gadget file**

   ```typescript title="gadgets/arcade.ts"
   import { Gadget, z } from 'llmist';

   export default class ArcadeHighScore extends Gadget({
     description: 'Check or record high scores on the arcade cabinet',
     schema: z.object({
       game: z.enum(['pac-man', 'galaga', 'donkey-kong']),
       action: z.enum(['check', 'record']),
       initials: z.string().length(3).optional(),
       score: z.number().optional(),
     }),
   }) {
     execute(params: this['params']): string {
       if (params.action === 'check') {
         return `High scores for ${params.game}: AAA-999999, BBB-888888, CCC-777777`;
       }
       return `${params.initials} recorded ${params.score} on ${params.game}!`;
     }
   }
   ```

3. **Run the agent with your gadget**

   <Tabs>
     <TabItem label="bun">
       ```bash
       bunx @llmist/cli agent --gadgets ./gadgets/arcade.ts \
         "Check the high scores for Pac-Man"
       ```
     </TabItem>
     <TabItem label="npm">
       ```bash
       npx @llmist/cli agent --gadgets ./gadgets/arcade.ts \
         "Check the high scores for Pac-Man"
       ```
     </TabItem>
   </Tabs>

</Steps>

Your directory structure:

<FileTree>
- gadgets/
  - arcade.ts
- package.json
</FileTree>

## Piping and Automation

The CLI integrates with Unix pipelines:

<Tabs>
  <TabItem label="bun">
    ```bash
    # Summarize a file
    cat README.md | bunx @llmist/cli complete "Summarize this document"

    # Code review
    cat src/main.ts | bunx @llmist/cli complete "Review this code for bugs"

    # Process multiple files
    find src -name "*.ts" -exec cat {} \; | \
      bunx @llmist/cli complete "Find potential security issues"
    ```
  </TabItem>
  <TabItem label="npm">
    ```bash
    # Summarize a file
    cat README.md | npx @llmist/cli complete "Summarize this document"

    # Code review
    cat src/main.ts | npx @llmist/cli complete "Review this code for bugs"

    # Process multiple files
    find src -name "*.ts" -exec cat {} \; | \
      npx @llmist/cli complete "Find potential security issues"
    ```
  </TabItem>
</Tabs>

## Choosing a Model

Specify the model with `--model` or `-m`:

<Tabs>
  <TabItem label="bun">
    ```bash
    # Use Claude Sonnet
    bunx @llmist/cli complete -m sonnet "Explain quantum computing"

    # Use GPT-5
    bunx @llmist/cli complete -m gpt5 "Write a haiku about coding"

    # Use Gemini Flash (fast and cheap)
    bunx @llmist/cli complete -m flash "What is 2 + 2?"
    ```
  </TabItem>
  <TabItem label="npm">
    ```bash
    # Use Claude Sonnet
    npx @llmist/cli complete -m sonnet "Explain quantum computing"

    # Use GPT-5
    npx @llmist/cli complete -m gpt5 "Write a haiku about coding"

    # Use Gemini Flash (fast and cheap)
    npx @llmist/cli complete -m flash "What is 2 + 2?"
    ```
  </TabItem>
</Tabs>

<Aside>
  Model aliases like `sonnet`, `gpt5`, and `flash` are shortcuts for full model names.
  See [Models & Aliases](/reference/models/) for all available models.
</Aside>
